# OpenClaw-TZ

太子工作区（Taizi Workspace）是一个基于 OpenClaw 的个人助理/多代理协作仓库，用于承接消息分拣、任务传达、知识沉淀、技能扩展与本地自动化工作。

本仓库当前偏向**自用部署**与**持续演进**，既可作为个人 AI 助理工作区，也可作为多角色代理协同的示例工程。

## 项目定位

该仓库主要用于：

- 作为 OpenClaw 中 `taizi` 代理的工作目录
- 维护人格、协议、用户信息与记忆文件
- 挂载与管理一批 Agent Skills
- 保存报告脚本、演示材料、截图和项目草稿
- 与其他代理（如中书省、尚书省等）形成分工协作

## 目录结构

```text
.
├── AGENTS.md          # 工作协议
├── SOUL.md            # 人设/角色说明
├── USER.md            # 用户信息
├── IDENTITY.md        # 身份设定
├── HEARTBEAT.md       # 心跳任务
├── memory/            # 记忆与工作记录
├── skills/            # 本地安装的技能
├── reports/           # 报告脚本与输出物
├── hr2/               # 业务项目/样例项目
├── edict/             # 独立项目（当前为嵌套仓库）
└── .openclaw/         # 本地状态文件（通常不纳入版本控制）
```

## 当前能力

本工作区已包含以下典型能力：

- Telegram 私聊接入
- 多代理角色分工
- 本地文件读写与命令执行
- 技能化扩展（文档、表格、PPT、PDF、前端、自动化等）
- 任务记录与阶段性回奏风格
- 与 GitHub 仓库联动进行版本管理

## 适用场景

- 搭建自己的 OpenClaw 智能助理
- 管理一个带人格与记忆的长期工作区
- 让多个代理角色承担不同职能
- 维护自己的技能库与自动化脚本
- 作为 OpenClaw 中文实战样例仓库使用

## 快速开始

请先阅读：

- [DEPLOYMENT.md](./DEPLOYMENT.md) —— 部署与启动流程

如果你已经有 OpenClaw 运行环境，只需把本仓库放到合适目录，并在 OpenClaw 配置中将 `taizi` 的 `workspace` 指向此目录即可。

## 注意事项

1. 本仓库中可能包含多个实验性目录，部署前建议先按自己的用途做一次清理。
2. `edict/` 当前是独立 git 仓库，如需统一管理，建议单独处理是否保留为子模块。
3. `.openclaw/`、认证文件、锁文件、日志、截图、构建产物等通常不建议直接纳入公开仓库。
4. 若要对外发布，建议补充脱敏、许可证、版本说明与架构图。
